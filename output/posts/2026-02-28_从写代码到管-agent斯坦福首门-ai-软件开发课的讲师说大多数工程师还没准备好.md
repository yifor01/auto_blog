# 📌 【斯坦福 CS146S 課程洞察】從寫程式碼到管 Agent，你準備好了嗎？

AI 時代的軟體開發正在經歷巨變。斯坦福大學在 2025 年秋季首次開設 CS146S「The Modern Software Developer」課程，這是全美第一門把 AI 貫穿整個軟體開發流程的學術課程。課程開放幾小時就吸引超過 100 名學生搶著報名。

但這門課的講師 Mihail Eric 認為：大多數工程師還沒有準備好迎接這場變革。

🤔 **初級開發者正面臨「三重風暴」**

Mihail 從一個真實故事說起：他認識一位剛從 Berkeley 畢業的工程師，投了約 1000 份履歷，只收到 2 個回覆——不是面試邀請，只是回覆。

為什麼會這樣？他歸納出三個同時發生的因素：

- **裁員潮**：COVID 後企業瘋狂擴招，後來發現砍掉 20-30% 員工業務照樣轉
- **畢業生暴增**：CS 畢業生數量過去十年翻了兩到三倍
- **AI 策略轉變**：雇主開始算「少招人+AI」的帳，用更少人完成同樣工作

這三者疊加，讓新人求職極其困難。Mih 認為，當前一代進入職場的初級開發者將是「AI 原生轉型」的第一代，他們必須同時具備傳統基本功和完整的 AI 原生能力。

🧪 **管理多個 Agent 是遊戲最終 BOSS**

課堂上，Claude Code 創始工程師 Boris Cherny 分享了他同時管理 10 多個 Agent 的工作方式。學生們的第一反應是：「我也應該從 10 個開始！」

Mihail 認為這是錯誤的。

**他的建議是「逐步搭建」**：
- 先用一個 Agent 把一件複雜的事做好
- 確認完全掌握這個工作流
- 再加一個處理隔離的小任務（如修 logo）
- 每個任務之間必須互相獨立、沒有依賴
- 不是 10 個一起上

真正會管理多個 Agent 的人，即使在今天也是最頂尖的那 0.1%。

⚠️ **什麼樣的程式碼庫對 Agent 友好？**

Mihail 指出，Agent 只能基於顯式「合約」（測試）運作，程式碼庫需要在以下三方面做好準備：

1. **測試覆蓋率高**：Agent 需要明確的測試作為依據
2. **文件一致性**：註解和文件必須準確反映程式碼
3. **設計模式一致性**：程式碼結構必須符合預期模式

💡 **品味在「最後一公里」培養**

有趣的是，連 Claude Code 團隊自己都在每周重寫程式碼。Mihail 認為，程式設計的品味和判斷力，仍然需要人類在長期實踐中培養。

他觀察到一個現象：資深開發者往往最抗拒 AI 工具，因為他們有歷史包袱；而初級工程師因為沒有包袱，反而學得最快。

🎯 **初級開發者的「無知無畏」是優勢**

Mihail 認為，初級開發者的「無知無畏」在 AI 時代反而成為優勢。他們更願意嘗試新工具、更不會被傳統思維束縛、更能快速適應變化。

但他強調：AI 原生不等於放棄基礎。傳統程式設計、系統設計、演算法思維的底子仍然要扎實。
