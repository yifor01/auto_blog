📌 【ETH Zurich x 巴塞尔大学 最新研究】Agent 仅靠论文方法复现社科结果

复现社科论文一定要原始代码？这个常识被 AI Agent 打破了。
ETH Zurich 团队实验证明：仅靠论文方法描述和原始数据，Agent 能复现多数已发表结果。
但失败的主因往往不是 AI 能力不足，而是论文本身描述不充分。

🤔 **复现社科结果长期依赖原始代码，AI 能否突破限制？**
过去利用 LLM Agent 复现社会科学实证结果的研究，通常会同时提供原始数据与作者公开代码，更接近“跑通现有代码”而非真实复现场景。本研究进一步拓宽边界，提出核心问题：如果仅提供论文的方法描述与原始数据，完全不提供原始代码、甚至不暴露原始结果与完整论文，Agent 能否独立完成结果复现？社会科学研究长期受可复现性困扰，多数研究因未附代码或代码描述不清难以验证，本研究的测试场景更贴近人类研究者的真实复现流程，对科学自动化的落地有重要参考价值。

🧪 **48篇已验证论文，4模型×4脚手架严格隔离测试**
研究团队开发了完整的 Agent 复现系统，流程包含三个核心环节：首先从论文中提取结构化的方法描述；随后在严格信息隔离环境下运行复现任务，Agent 仅能获取结构化方法描述与原始数据，完全无法接触原始代码、原始结果或完整论文内容；最后通过确定性的单元格级（cell-level）比对，将复现结果与原始结果逐层级校验，并加入错误归因步骤，可沿着系统链路追溯差异的根本原因。

评估环节共测试4种 Agent 脚手架（scaffolds）、4种大语言模型，覆盖48篇经过人工验证可复现的社会科学论文，确保基线结果可靠。

🔍 **多数结果可被复现，性能差异跨度极大**
评估结果显示，Agent 整体可恢复大部分已发表的社会科学结果，但性能表现存在高度异质性：不同 LLM 之间、不同 Agent 脚手架之间、不同论文之间的复现成功率差异显著。

根因分析进一步揭示，复现失败的原因并非仅来自 Agent 的执行错误，论文本身的方法描述不充分（underspecification）同样是核心因素，比如方法章节未明确数据预处理规则、统计参数设置等细节，会直接导致 Agent 复现偏差。

💡 **复现失败双主因，论文质量也是关键变量**
过往类似研究多将复现失败归咎于 AI 能力短板，本研究的错误归因机制首次明确将论文本身的描述质量纳入失败根因。这意味着这套系统不仅能评估 Agent 的科学复现能力，也可反向用于检验社会科学论文的可复现性质量，帮助作者补全方法章节的疏漏，从流程上提升社科研究的规范性。

⚠️ **仅测已验证可复现论文，未覆盖不可复现场景**
当前研究存在明确的边界限制：首先评估样本为48篇已通过人工验证可复现的论文，未包含本身存在可复现性问题的研究，结果可能高估真实场景下的复现成功率；其次仅测试了4种 LLM 与4种 Agent 脚手架，未覆盖当前主流的所有模型与框架，结论的泛化性有待进一步验证；此外严格信息隔离是理想化设置，真实研究中复现者可能会参考更多补充材料，场景差异可能影响结果表现。

🎯 **可扩展评估框架，同时服务AI与社科研究者**
对于 GenAI 工程师与研究者，这套系统提供了标准化的 Agent 科学复现能力评估框架，搭配错误归因链路，可高效定位 Agent 的执行短板，为科学自动化场景的 Agent 优化提供清晰思路。

对于社会科学研究者，可借助这套系统提前检验论文方法描述的充分性，在投稿前补全疏漏，从流程上提升研究可复现性。

整体而言，本研究明确了当前 Agent 在科学复现任务中的能力边界：既需要更鲁棒的 Agent 设计，也需要更规范的学术写作标准，二者共同推进科学自动化的落地。
