📌 為 Agent 設計產品：80% 互動不再經過 UI  

Salesforce 在 27 年來最激進的架構轉型中公開承認：未來大量使用行為將透過 Claude、ChatGPT 與後台流程完成，並主動把整個平台拆解為 API、MCP 工具與 CLI。當 AI Agent 能推理、規劃與執行，傳統圖形介面是否還是必選？這篇來自 寶玉 baoyu.io 的深度觀察，試圖回答這個正在改寫 B2B 軟體邊界的問題。  

🧩 界面不死，但二八法則已經顛倒  

過去二十年，人與軟體的互動路徑很單一：使用者 → 介面 → 資料庫。介面幾乎等同於產品本身，也是使用者建立信任與完成確認的關鍵。然而，當 Agent 開始代理點擊、瀏覽、讀寫，介面不再是唯一通道。預估未來 80% 的人機互動將透過 Agent 完成，不僅改變「做什麼」，更決定「怎麼做」。  

🛠️ 52 倍成長的 MCP 週活，揭示入口遷移  

以 Ramp 為例，過去三個月隨著越來越多的客戶透過 Claude、ChatGPT 等 Agent 進入產品，其 MCP（Model Context Protocol，模型上下文協議）上的週活成長 10 倍。MCP 作為讓 Agent 標準化呼叫外部工具與資料的協議，正快速成為新的交互層。當 Agent 能直接對話底層系統，瀏覽器介面便從必要條件退守為可選通道。  

⚡ Salesforce 拆解自己，把能力全部暴露為 API 與 MCP  

在 TDX 開發者大會上，Salesforce 推出 Headless 360，一次將平台所有能力開放為 API、MCP 工具與 CLI 命令。這不僅是架構上的「無頭化」，更是對傳統 CRM 必須依賴圖形介面的質疑。面對「如果 Agent 能執行，還需要一個帶 UI 的 CRM 嗎？」這道生死題，Salesforce 的選擇是：不需要——並以 100 多個新工具與技能向開發者證明這一點。  

🤝 雙 Agent 協作：使用者 Agent 對上軟體自身的 Agent  

未來的互動模型逐漸從單層代理走向雙層協作：使用者的 Agent 提出意圖，軟體自身的 Agent 負責處理業務邏輯、落實規則、補足缺失上下文。兩個大型語言模型（LLM）各自扮演專家角色，在同一目標下分工推進。這不僅降低使用者的操作負擔，也讓軟體得以在無 UI 情境下維持一致性與可靠性。  

⚠️ 信任、確認與長期 UX 的權衡仍未解決  

儘管入口遷移至 Agent，報告仍坦承 UI 並未死亡。人類依舊需要點擊、檢視設定、確認任務完成，特別是在高風險或高價值的企業場景。此外，Agent 間的協作規範、錯誤修復的可追蹤性，以及長期學習與治理成本，目前多數仍處於早期實踐階段，長期穩定性有待更多產業驗證。  

🎯 從「幫我寫」轉向「讓 Agent 替我執行」的設計思維  

- 將核心能力從 UI 中解耦，優先暴露為 API、MCP 與 CLI，是未來 B2B 產品的基礎工程  
- 設計軟體自身的 Agent 角色，使其能與外部 Agent 協商業務規則與上下文  
- 保留輕量 UI 專注於確認、審查與例外處理，而非日常執行  
- 評估 MCP 等標準化協議作為新交互層的長期可維護性與權限模型
