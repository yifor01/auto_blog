📌 低成本合成數據訓練 Search Agent

訓練能進行深度研究的 Search Agent，過去總被海量人工標註成本綁架。這篇研究提出一套「零付費 API」的合成數據管線，卻讓 4B 小模型在多步檢索任務上展現越級表現。

🤔 **多步推理數據稀缺，Agent 訓練遭遇瓶頸**
當語言模型結合網頁搜尋成為解答複雜問題的標配，Agent 的訓練面臨一個結構性難題：高品質的多步檢索與推理數據極度稀缺。傳統方法高度仰賴昂貴的人工標註，或是需要繁瑣的前置環境設定，這使得開發者難以在有限預算下擴充訓練集。如何在控制成本的前提下，產出足以支撐深度推理的訓練資料，成為推動 Search Agent 技術落地的關鍵。

🧪 **四階段模組化管線，內建自我與外部雙重驗證**
研究團隊提出 ORBIT 框架，以模組化設計解決上述痛點。管線分為四個階段：種子查詢生成 (seed creation)、問答對生成、自我驗證 (self-verification) 以及外部驗證 (external verification)。該流程完全不依賴付費 API 服務，專注於生成 20,000 筆具備簡短可驗證答案的密集推理查詢。數據橫跨 15 個領域，每筆訓練配對需經過 4 到 5 個推理步驟，且最終階段強制要求透過完整網頁進行外部搜尋驗證，確保答案的準確性與邏輯閉環。

📊 **ORBIT-4B 性能躍升，子 4B 模型展現高性價比**
研究團隊將 Qwen3-4B 作為基底模型，使用 ORBIT 數據集搭配 GRPO (Group Relative Policy Optimization，一種基於群體相對優勢的強化學習對齊演算法) 進行微調。在維基百科問答任務的廣泛評估中，ORBIT-4B 展現了強勁的競爭力，在參數量低於 4B 的搜尋 Agent 模型中名列前茅。這項結果直接證明：只要數據具備足夠的推理密度與嚴謹的驗證機制，合成數據的訓練效用足以媲美高成本標註。

💡 **合成數據的價值，在於可驗證的推理路徑而非數量**
ORBIT 的核心洞察在於將「可驗證性」置於數據生成的中心。不同於過去盲目擴大合成數據規模的做法，該管線透過雙重驗證機制過濾幻覺與邏輯斷層。自我驗證確保模型內部的推理一致性，外部網頁驗證則將答案錨定於真實世界資訊。結合 GRPO 的優化特性，模型能在多步檢索中學習到更穩定的決策路徑。這顯示 Agent 訓練的瓶頸不在於數據量，而在於數據的邏輯嚴謹度與可驗證性。

⚠️ **驗證依賴外部搜尋，評估範圍限於維基百科 QA**
儘管框架設計完整，研究仍存在幾項限制。外部驗證階段高度依賴完整網頁的搜尋結果，這可能引入網頁結構變動或搜尋引擎排序偏差的干擾。此外，實驗評估主要聚焦於維基百科問答任務，尚未全面涵蓋需要跨多源動態抓取或處理長時序互動的複雜現實場景。對於需要極端即時性或高度領域專業知識的應用，其泛化能力仍待進一步驗證。

🎯 **開源小模型微調新範式，Frugal Pipeline 值得複用**
對於工程實踐而言，ORBIT 提供了一套可直接套用的低成本數據生產標準。建議開發者在自建 Agent 訓練集時，優先導入「可驗證答案」的設計原則，並參考其四階段管線進行數據清洗。小模型結合 GRPO 的微調路徑已證實可行，團隊可將資源集中於優化檢索召回與驗證邏輯，而非盲目擴充模型規模。該研究的程式碼與數據集已全面開源，極適合作為入門多步推理 Agent 的實作基底。
